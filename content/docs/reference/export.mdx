## Export Formats

DexCode supports multiple export formats for different use cases. Export is available through the browser UI when viewing a deck.

## PDF Export

Generate a PDF of your deck directly from the browser toolbar. Each slide is rendered as a high-resolution JPEG image and combined into a PDF using jsPDF.

### How It Works

1. Each slide is captured at full resolution using html-to-image
2. The DOM waits for images, fonts, and animations to stabilize before capture
3. Slides are rendered as JPEG images and inserted into a PDF
4. The output is downloaded via the browser

<Callout type="info">
PDF export runs entirely in the browser — no server-side rendering or Playwright installation is required.
</Callout>

## PPTX Export

Export to PowerPoint format from the browser toolbar. Each slide is rendered as a high-resolution image and inserted into a PPTX file using pptxgenjs.

<Callout type="info">
PPTX export creates image-based slides. Interactive elements (animations, hover effects) are captured as static snapshots.
</Callout>

## Static Site Deployment

Build the project as a standard Next.js application:

```bash
npm run build
```

The built output can be deployed to any hosting service that supports Next.js (Vercel, etc.).

## Live Sharing via Cloudflare Tunnel

For real-time sharing, use Cloudflare Tunnel to expose your local dev server:

```bash
npx cloudflared tunnel --url http://localhost:3000
```

Or ask your AI agent:

```bash
> Expose this deck via Cloudflare Tunnel and share the URL
```

This generates a public URL that anyone can access. Viewers see your slides in real time — any updates you make are reflected immediately.

### Use Cases

- Real-time deck review with remote team members
- Live presentation to external audiences
- Quick sharing without deployment
