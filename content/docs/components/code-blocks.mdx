## Code Blocks

DexCode uses **Shiki** for syntax highlighting with the One Dark Pro theme. Code blocks are automatically styled to match your deck's dark theme.

## Basic Usage

Use standard Markdown fenced code blocks with a language identifier:

```typescript
function greet(name: string): string {
  return `Hello, ${name}!`;
}
```

## Supported Languages

DexCode supports 100+ programming languages through Shiki, including:

- TypeScript / JavaScript
- Python
- Rust
- Go
- Java
- C / C++
- Ruby
- Swift
- SQL
- Bash / Shell
- HTML / CSS
- JSON / YAML / TOML

## Multi-Language Code

Show code in multiple languages using the Tabs component. Each tab contains a code block in a different language, letting viewers switch between implementations.

### TypeScript Example

```typescript
const result = await fetch("/api/data");
const data = await result.json();
```

### Python Example

```python
import requests
result = requests.get("/api/data")
data = result.json()
```

### Go Example

```go
resp, err := http.Get("/api/data")
defer resp.Body.Close()
data, _ := io.ReadAll(resp.Body)
```

## Terminal Blocks

For shell commands, use the Terminal component to render a styled terminal window:

```bash
$ git clone git@github.com:co-r-e/dexcode.git
$ cd dexcode && npm install
$ npm run dev
```

## Code with Explanations

Pair code with descriptive text using the Columns component. Place the code block on one side and explanation on the other for a clean layout.

```typescript
// Define your slide configuration
export default defineConfig({
  title: "Sprint Review",
  slides: 12,
  theme: "default",
});
```

The configuration file controls deck metadata, slide count targets, and visual theme selection. All settings can be overridden per-slide in the MDX frontmatter.

<Callout type="info">
Code blocks include a copy-to-clipboard button that appears on hover. This works in both preview and presenter mode.
</Callout>
